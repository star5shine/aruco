# ArUco 网络摄像头识别系统使用说明

## 1. 文件说明

程序目录建议放在：

```bash
/root/aruco
```

主要文件：

```text
aruco_desktop_ui.py       桌面 UI 主程序
aruco_netcamera.py        原始识别程序，可单独运行
netcamera_calib.npz       网络摄像头标定内参文件
capture_calib_images.py   标定图片采集程序
calibrate_netcamera.py    网络摄像头标定程序
install_autostart.sh      桌面自启动安装脚本，可选
```

当前 UI 默认读取同目录下的：

```text
/root/aruco/netcamera_calib.npz
```

## 2. 环境要求

需要安装：

```bash
sudo apt install python3-tk ffmpeg
```

Python 需要能导入：

```python
cv2
numpy
```

如果 `python3 aruco_desktop_ui.py` 报：

```text
ModuleNotFoundError: No module named 'tkinter'
```

说明缺少 Tkinter，安装：

```bash
sudo apt install python3-tk
```

## 3. 启动 UI

进入程序目录：

```bash
cd /root/aruco
python3 aruco_desktop_ui.py
```

如果在板子本机屏幕显示，必要时先设置：

```bash
export DISPLAY=:0
python3 aruco_desktop_ui.py
```
## 4.查询网络摄像头IP

1.连接板子WiFi，查看电脑IP，（win+r输入cmd，打开终端，输入ipconfig查看电脑ip）
2.打开海康交付助手，刷新查看摄像头IP，如果IP第3位与电脑不一致，改成与电脑一致
例：电脑ip：192.168.1.105
摄像头ip：192.168.0.10
摄像头ip前方框打勾，在弹出的右边栏将摄像头IP改成192.168.1.10，本地网关改成192.168.1.1，密码为12345(一般情况下)
3.摄像头连接板子，板子ping摄像头ip，如果没有ping通，再打开海康威视交付助手重新修改

## 5. 使用流程

1. 打开 UI：

```bash
cd /root/aruco
python3 aruco_desktop_ui.py
```

2. 填写摄像头 IP、TCP IP 和端口。

3. 点击 `Save` 保存配置。

4. 点击 `Start` 开始识别。

5. 点击 `Stop` 停止识别。

识别成功后，画面中会显示 ArUco 框、坐标和角度。

## 6. TCP 报文格式

当前只发送一种报文类型：

```text
QR_PARKING
```

发送格式为 JSON，每条报文一行：

```json
{
  "type": "QR_PARKING",
  "seq": 1,
  "timestamp_ms": 1710000000000,
  "valid": true,
  "confidence": 1.0,
  "right_m": 0.02,
  "forward_m": 1.25,
  "yaw_error_deg": -1.8
}
```

字段说明：

```text
type            固定为 QR_PARKING
seq             报文序号，从 1 开始递增
timestamp_ms    当前时间戳，单位毫秒
valid           检测到 ArUco 时为 true
confidence      当前固定为 1.0
right_m         右向偏移，单位米，对应原 X
forward_m       前向距离，单位米，对应原 Z
yaw_error_deg   偏航角误差，单位度，对应原 Angle
```

## 7. 开机自启动

开机自启动使用 systemd 服务方式。这个方法不只适用于当前 RK3588 板子，其他 Linux 板子也可以使用，只要满足：

```text
1. 系统使用 systemd
2. 板子有图形桌面环境
3. 程序目录为 /root/aruco，或按实际目录修改服务文件
4. python3、python3-tk、ffmpeg、opencv、numpy 已安装
```

### 7.1 新板子安装自启动

在目标板子上创建服务文件：

```bash
cat > /etc/systemd/system/aruco-ui.service <<'EOF'
[Unit]
Description=ArUco Desktop UI
After=graphical.target
Wants=graphical.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/aruco
Environment=DISPLAY=:0
Environment=XAUTHORITY=/root/.Xauthority
ExecStartPre=/bin/sleep 8
ExecStart=/usr/bin/python3 /root/aruco/aruco_desktop_ui.py
Restart=no

[Install]
WantedBy=graphical.target
EOF
```

启用并启动：

```bash
systemctl daemon-reload
systemctl enable aruco-ui.service
systemctl start aruco-ui.service
```

重启测试：

```bash
reboot
```

### 7.2 查看状态和日志

查看服务状态：

```bash
systemctl status aruco-ui.service
```

查看日志：

```bash
journalctl -u aruco-ui.service -n 80 --no-pager
```

### 7.3 手动控制

手动启动：

```bash
systemctl start aruco-ui.service
```

手动停止：

```bash
systemctl stop aruco-ui.service
```

重启服务：

```bash
systemctl restart aruco-ui.service
```

取消开机自启动：

```bash
systemctl disable aruco-ui.service
systemctl stop aruco-ui.service
```

重新启用：

```bash
systemctl enable aruco-ui.service
```

### 7.4 注意事项

服务中使用：

```ini
Restart=no
```

这样开机只自动启动一次。手动关闭 UI 后，不会被 systemd 自动重新拉起。

如果希望程序异常退出后自动重启，可以改成：

```ini
Restart=on-failure
```

不建议使用：

```ini
Restart=always
```

因为这样你手动关闭 UI 后，它也会重新弹出来。

如果其他板子的桌面不是 root 用户登录，可能需要把服务里的这几项改成实际桌面用户：

```ini
User=root
Environment=XAUTHORITY=/root/.Xauthority
```

例如桌面用户是 `linaro`，可能需要改成：

```ini
User=linaro
Environment=XAUTHORITY=/home/linaro/.Xauthority
WorkingDirectory=/home/linaro/aruco
ExecStart=/usr/bin/python3 /home/linaro/aruco/aruco_desktop_ui.py
```

如果 UI 没有显示，先确认显示号：

```bash
echo $DISPLAY
```

常见本机屏幕是：

```text
:0
```

如果不是 `:0`，需要修改服务里的：

```ini
Environment=DISPLAY=:0
```

## 8. 网络摄像头标定

标定图片保存目录：

```text
/root/aruco/calib_images
```

采集标定图片：

```bash
cd /root/aruco
python3 capture_calib_images.py
```

按 `s` 保存图片，按 `q` 或 `ESC` 退出。

运行标定：

```bash
cd /root/aruco
python3 calibrate_netcamera.py
```

生成标定文件：

```text
/root/aruco/netcamera_calib.npz
```

注意：标定图片分辨率应尽量和识别分辨率一致，否则距离结果可能不准。

## 9. 常见问题

### 9.1 UI 没有画面

先确认摄像头网络是否可达：

```bash
ping 192.168.1.11
```

### 9.2 No route to host

表示板子连不到摄像头 IP。

检查：

```bash
ip addr
ip route
ping 摄像头IP
```

确认板子和摄像头在同一网段，或者路由可达。

### 9.3 TCP connect failed / timed out

说明 TCP 接收端没有打开，或者 IP/端口不通。

如果暂时不用 TCP，把 UI 中 `TCP IP` 留空。

### 9.4 UI 延迟比原始程序高

桌面 UI 需要额外显示图像，所以会比 `aruco_netcamera.py` 略高。

降低延迟建议：

```text
Width  = 800
Height = 448 或 450
TCP IP 留空，确认不是 TCP 超时导致卡顿
优先使用本机屏幕 DISPLAY=:0
```

### 9.5 标定结果差距大

检查：

```text
1. 棋盘格内角点数量是否正确
2. 方格尺寸 square-size 是否正确
3. 有效图片是否至少 20 张
4. 图片是否覆盖不同位置和角度
5. 标定分辨率和识别分辨率是否一致
```

## 10. 单独运行原始识别程序

如果不使用 UI，也可以直接运行：

```bash
cd /root/aruco
```
